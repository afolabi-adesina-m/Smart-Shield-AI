# Smart-Shield Project — File & Folder Guide

This document explains **every folder and file** in the Ontario Smart-Shield capstone repository so teammates, professors, and future you can navigate the project quickly.

**Course:** INFO53883 AI & ML Capstone · **Team:** 2B · **Sheridan College · Spring 2026**

---

## How the project fits together

```
Data/  ──►  notebooks/  ──►  models/  ──►  demo/
                ▲                              │
                └──────── src/ ◄───────────────┘
```

1. **Data** — raw collision CSVs and road-condition images  
2. **notebooks** — full analysis, modelling, and evaluation pipeline  
3. **src** — reusable Python modules (NLP, vision, safety score)  
4. **models** — trained artifacts saved from the notebook  
5. **demo** — live map web app that loads models and scores routes

## `notebooks/` — Main analysis pipeline

| File | Purpose |
|------|---------|
| `capstone.ipynb` | **Active working notebook.** Run top-to-bottom for EDA, feature engineering, model training, SHAP, and deployment. Works on local PC, Colab, or Kaggle (setup cell auto-detects). |
| `capstone_with_results.ipynb` | **Annotated copy with saved outputs.** Same pipeline as `capstone.ipynb`, plus: basic descriptive statistics, a *"What this cell does"* block before each code cell, and *"Findings & importance"* markdown after each output. Use this for presentations and reports. |

**Typical workflow:** Environment setup → load data → EDA → preprocessing → correlation & feature selection → vision fine-tuning → SMOTE + baselines → GridSearch → DNN → multimodal fusion → save models.

## `Data/` — Datasets and vision cache

| File / folder | Purpose |
|---------------|---------|
| `README.md` | Lists required CSV filenames and where to download UK DfT data |
| `traffic collision data.csv` | **Toronto Police Service** collision records (~809K rows). Primary tabular dataset for severity classification. |
| `dft-road-casualty-statistics-collision-2024.csv` | **UK Department for Transport** 2024 collision records (~101K rows). Used for weather/surface E_index calibration (Paper 2 replication). |
| `vision_cache/` | Sample road-surface JPEG images for the Vision Brain (offline training when live cameras are unavailable) |

### `Data/vision_cache/` subfolders

| Subfolder | Content |
|-----------|---------|
| `clear/`, `dry_asphalt_smooth/`, `dry_asphalt_slight/` | Clear / dry asphalt reference images |
| `wet/`, `wet_asphalt_smooth/`, `wet_asphalt_slight/` | Wet pavement images |
| `snow/`, `fresh_snow/` | Snow-covered road images |
| `ice/` | Ice / black-ice surface images |

Images are from Ontario highway RWIS-style sources (UWaterloo iTSS Lab naming). `demo_00.jpg` files are synthetic placeholders for quick demos.

**Optional CSVs** (listed in `Data/README.md` but not required for the core pipeline): additional DfT casualty/vehicle files, Seattle SDOT collisions.

## `models/` — Trained artifacts

Generated by running the modelling sections of `capstone.ipynb`. The demo loads these at startup.

| File | Purpose |
|------|---------|
| `README.md` | Short index of model files |
| `rf_tuned.joblib` | **Production tabular model** — tuned Random Forest classifier for collision severity |
| `lr_tuned.joblib` | Tuned Logistic Regression (baseline / comparison) |
| `scaler.joblib` | `StandardScaler` fitted on training features — must be applied before inference |
| `feature_names.joblib` | Ordered list of 8 input feature names (hour, season, pedestrian, etc.) |
| `tfidf_vectorizer.joblib` | TF-IDF vectorizer for Ontario 511-style alert text (NLP Brain) |
| `dnn_smart_shield.pt` | PyTorch deep neural network weights (tabular DNN pillar) |
| `vision_resnet18.pt` | Fine-tuned ResNet18 weights for road-surface classification (Vision Brain) |

> Large `.joblib` / `.pt` files are gitignored. Retrain locally or use Git LFS to share.

## `src/` — Reusable Python modules

Imported by the notebook and `demo/`. Keeps the 3-Brain architecture modular.

| File | Purpose |
|------|---------|
| `__init__.py` | Marks `src` as a Python package |
| `nlp_brain.py` | **NLP Brain (Pillar 1):** TF-IDF scoring of hazard terms in Ontario 511-style alerts; scenario text for live test cases TC-1…TC-5 |
| `vision_brain.py` | **Vision Brain (Pillar 2):** ResNet18 loading, image preprocessing, road-condition class probabilities (clear / wet / snow-ice) |
| `safety_score.py` | **Fusion engine:** combines Text (T), Vision (V), and Environment (E) scores into composite Safety Score **S**; builds speed advisories |
| `cm_helpers.py` | Plotting helpers for side-by-side confusion matrices (raw counts + normalized) |

## `explanations/` — Auto-generated documentation

Word docs and diagrams that stay in sync with the notebook and project metadata. **Do not edit the `.docx` files by hand** — edit `definitions.json` and regenerate.

| File | Purpose |
|------|---------|
| `definitions.json` | **Single source of truth** — glossary terms, sprint lanes, split constants |
| `build_all.py` | Master builder — scans notebook/Data, writes all outputs |
| `build_docs.py` | Thin wrapper → calls `build_all.py` (legacy entry point) |
| `update.bat` | Windows one-click rebuild |
| `Glossary.docx` | 60+ ML/project term definitions |
| `Sprint-Pipeline-Swimlane.docx` | Sprint pipeline tables (9 lanes, 25 steps) |
| `Sprint-Pipeline-Swimlane.mmd` | Mermaid source — render at [mermaid.live](https://mermaid.live) |
| `Train-Test-Validation-Splits.docx` | 80/20 stratified split, SMOTE, 3-fold CV reference |
| `README.md` | Index with last-built timestamp (auto-generated) |
| `MANIFEST.json` | Source mtimes and output list (auto-generated) |

### Automatic rebuild triggers

1. **Cursor hook** — `.cursor/hooks.json` runs `update-explanations.ps1` after saving `notebooks/*.ipynb` or `definitions.json`
2. **Git pre-commit** — `.githooks/pre-commit` rebuilds and stages docs when those files are committed (enable: `git config core.hooksPath .githooks`)
3. **Watch mode** — `python explanations/build_all.py --watch` (30s polling)

## Quick reference — which file do I need?

| I want to… | Open this |
|------------|-----------|
| Run the full analysis | `notebooks/capstone.ipynb` |
| View results with explanations | `notebooks/capstone_with_results.ipynb` |
| Run notebook + auto-save + update docs | `run_notebook.bat` or **Terminal → Run Build Task** (`Ctrl+Shift+B`) |
| Post-run update after Run All in editor | Last notebook cell runs automatically; or `python scripts/post_notebook_run.py` |
| Launch the live map demo | `demo/api_server.py` |
| Understand the Safety Score formula | `src/safety_score.py` |
| Add / replace training data | `Data/` (see `Data/README.md`) |
| Retrain and save models | Notebook Section 8 + 10.4 → `models/` |
| Prepare the presentation | `docs/FINAL-PRESENTATION-SLIDE-OUTLINE.md` + `assets/Final.png` |
| Glossary / pipeline / split docs | `explanations/` — edit `definitions.json`, run `build_all.py` |
| Track post-submission fixes | `improvements/speed-advisory-audit/` |
| Find an old notebook version | `archive/notebooks/` |
